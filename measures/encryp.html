เทคนิคการเข้ารหัส
  1. เป็นการเข้ารหัสข้อมูลแบบ block ciphers ตามวิธีมาตรฐานการเข้ารหัสขั้นสูง (Advanced Encryption Standrad : AES) ของ U.S. National Institute of Standards and Technology (NIST) ซึ่งได้มาตรฐาน ISO/IEC 18033-3 เมื่อปี พ.ศ. 2548

  2. รหัสจะเจาะได้ยาก ถ้า private key มีความยาว (จำนวนบิทมาก) โดยความยาวของ private key จะเป็นตัวกำหนด variants ให้มีขนาด 128 บิท 192 บิท หรือ 256 บิท แล้วแต่กรณี บิทยิ่งมาก การเข้ารหัสโดยผสมข้อมูลกับ round key ในลักษณะ nonlinear complex modification จะเพิ่มรอบมากขึ้น เป็น 10 12 และ 14 รอบตามลำดับ รายละเอียดตาม Github repository (crypto-js)

  3. ปัจจุบัน (พ.ศ. 2567) สำนักงานความมั่นคงปลอดภัยไซเบอร์และโครงสร้างพื้นฐานของสหรัฐอเมริกา (Cybersecurity and Infrastructure Security Agency) ยังคงแนะนำให้หน่วยงานรัฐบาลสหรัฐอเมริกา ใช้วิธี AES ในการเข้ารหัสข้อมูลราชการลับ

  4. คณะกรรมการด้านระบบความมั่นคงปลอดภัยแห่งชาติ (Committee on National Security Systems: CNSS) ของสหรัฐอมเริกา กำหนดนโยบายให้ใช้ AES เข้ารหัสข้อมูลราชการลับ ระดับ "TOP SECRET" โดยใช้ key ขนาด 192 หรือ 256 บิท ส่วน key ขนาด 128 บิท สามารถใช้เข้ารหัสข้อมูลราชการลับ ระดับ "SECRET"

  5. โปรแกรมนี้ เขียนด้วยภาษา HTML/JavaScript โดยเปิดเผย source code ไม่มีการบันทึกข้อมูลที่เข้ารหัสไว้แต่อย่างใด หากผู้เข้ารหัสลืม private key และได้ทำลายข้อมูลต้นฉบับแล้ว จะไม่สามารถกู้ข้อมูลต้นฉบับได้ เว้นแต่จะคาดเดา private key ได้อย่างถูกต้อง ดังนั้น สมควรกำหนด private key ที่ปลอดภัย เช่น การนำ password ที่ตนเองจำได้ง่าย มา hash เป็น private key โดยใช้ Secure Hash Algorithm (SHA) เช่น SHA3-512 เพื่อแปลง password เป็น private key
