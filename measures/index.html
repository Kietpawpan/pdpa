7 มาตรการรักษาความมั่นคงปลอดภัยของข้อมูล

โดย มนตรี เกียรติเผ่าพันธ์

การรักษาความมั่นคงปลอดภัยของข้อมูล คือ การธำรงไว้ซึ่งความลับ (confidentiality) ความถูกต้องครบถ้วน (integrity) และสภาพพร้อมใช้งาน (availability) ของข้อมูล ทั้งนี้ เพื่อป้องกันการสูญหาย เข้าถึง ใช้ เปลี่ยนแปลง แก้ไข หรือเปิดเผยข้อมูลโดยปราศจากอำนาจหรือโดยมิชอบ โดยเฉพาะข้อมูลดิจิทัลที่รั่วไหลได้ง่าย จะต้องมีมาตรการที่เหมาะสม อาทิ

  • การเข้ารหัส
  • การสำรอง
  • การจำกัดการเข้าถึง
  • การรักษาเครือข่ายสารสนเทศ
  • การติดตั้งซอฟแวร์รุ่นใหม่
  • การรักษาความปลอดภัยทางกายภาพ และ
  • การใช้มาตรการเชิงบริหาร

1. เข้ารหัส (Encryption)

แปลงข้อมูลเป็นรหัส ที่อ่านไม่ได้ และต้องใช้กุญแจส่วนตัว (decryption key หรือ private key) เพื่อถอดรหัสนั้น กลับเป็นข้อมูลต้นฉบับ ทั้งนี้ รัฐบาลกลางสหรัฐอเมริกาใช้วิธี Advanced Encryption Standrad (AES) ของสถาบันมาตรฐานและเทคโนโลยีแห่งชาติ (U.S. National Institute of Standards and Technology: NIST) ซึ่งได้มาตรฐาน ISO/IEC 18033-3 (มาตรฐานเทคโนโลยีสารสนเทศ - เทคนิคการรักษาความมั่นคงปลอดภัย : การเข้ารหัสแบบ block ciphers)

2. สำรองข้อมูล (Backup)

จัดทำสำเนาข้อมูล 3 ชุด เก็บไว้แยกอุปกรณ์ 2 ชุด อยู่ในที่ปลอดภัย อีกชุดหนึ่งเก็บไว้นอกสถานที่ เช่น บนคลาวด์ที่มีความมั่นคงปลอดภัย เพื่อป้องกันกรณีข้อมูลต้นฉบับสูญหายหรือถูกทำลายจนไม่อยู่ในสภาพพร้อมใช้งาน เช่น การใช้บริการระบบคลาวด์กลางภาครัฐหรือ Government Data Center and Cloud Service: GDCC ของกระทรวงดิจิทัลเพื่อเศรษฐกิจและสังคม ซึ่งมีการ backup ย้อนหลัง 7 วัน อนึ่ง หากเป็นข้อมูลที่เปิดเผยได้ อาจเก็บไว้บนคลาวด์เอกชน เช่น Github และ Google Dive เป็นต้น

3. จำกัดการเข้าถึง (Access Control)

กรณีข้อมูลกระดาษ กำหนดให้ข้อมูลส่วนบุคคลเป็นข้อมูลข่าวสารลับ ตามระเบียบว่าด้วยการรักษาความลับของทางราชการ พ.ศ. 2544 และที่แก้ไขเพิ่มเติม ให้เข้าถึงได้เฉพาะผู้มีหน้าที่เกี่ยวข้อง เท่าที่จำเป็นเท่านั้น กรณีข้อมูลดิจิทัล ให้มีระบบ login ด้วย username และ password ที่จำง่าย พิมพ์ง่าย เดายาก (Entropy) เจาะยาก และห้ามเปิดเผยหรืออนุญาตให้ผู้อื่นใช้โดยเด็ดขาด รวมถึงมีระบบ multi-factor authentication เช่น ให้กรอกรหัส OTP ที่ระบบส่งให้ทางอีเมลส่วนตัว

4. รักษาเครือข่ายให้มั่นคงปลอดภัย (Network Security)

จัดให้มีระบบรักษาความปลอดภัย เช่น Firewall, Application Firewall, Antivirus, DDoS Attack และแลกเปลี่ยนข้อมูลผ่านระบบ Secure Socket Layer (SSL Certificate) โดยหน่วยงานในสังกัดสำนักงานปลัดกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อมที่มีโดเมนย่อย *.mnre.go.th สามารถขอรับไฟล์ SSL Certificate ของหน่วยงานได้ที่ศูนย์เทคโนโลยีดิจิทัลและอากาศยาน และติดตั้งบนเครื่อง Server ของระบบที่ตนดูแล ถ้าใช้ระบบ Apache HTTP สามารถดูวิธีติดตั้งที่ https://github.com/Kietpawpan/Apache สำหรับการรักษาความมั่นคงปลอดภัยของเครือข่าย ควรศึกษาเรื่อง Cyber Security และประกาศคณะกรรมการการรักษาความมั่นคงปลอดภัยไซเบอร์แห่งชาติ เรื่อง ลักษณะภัยคุกคามทางไซเบอร์ มาตรการป้องกัน รับมือ ประเมิน ปราบปราม และระงับภัยคุกคามทางไซเบอร์แต่ละระดับ พ.ศ. 2564 อนึ่ง สำหรับการพัฒนาแอพพลิเคชันของหน่วยงาน ก็ควรปฏิบัติตามมาตรฐานที่เกี่ยวข้อง เช่น มาตรฐานแอปพลิเคชันภาครัฐสำหรับ อุปกรณ์เคลื่อนที่ เป็นต้น

5. ติดตั้งซอฟแวร์รุ่นใหม่

หมั่นตรวจสอบและติดตั้งซอฟแวร์ของเครื่องคอมพิวเตอร์แม่ข่ายและเครื่องคอมพิวเตอร์ที่ใช้งาน ให้เป็นเวอร์ชันใหม่ (update) ซึ่งมีเทคโนโลยีรักษาความมั่นคงปลอดภัยไซเบอร์ที่ดียิ่งขึ้น เช่น โปรแกรม PHP / Hash algorithm / Encryption Algorithm / Windows / iOS / Android / Apache

6. เชิงกายภาพ

รักษาความมั่นคงปลอดภัยของอุปกรณ์เก็บข้อมูลและสถานที่จัดเก็บอุปกรณ์ดังกล่าว ไม่ให้ถูกเข้าถึงโดยผู้ไม่มีหน้าที่เกี่ยวข้อง เช่น การเก็บข้อมูลในตู้เหล็กที่มีกุญแจล็อคและผู้ควบคุมกุญแจ สมุดบันทึกการเข้าถึงข้อมูล

7. เชิงบริหาร

กรณีข้อมูลส่วนบุคคล ผู้ควบคุมข้อมูลส่วนบุคคล จะต้องดำเนินการให้เป็นไปตามประกาศคณะกรรมการคุ้มครองข้อมูลส่วนบุคคล เรื่อง มาตรการรักษาความมั่นคงปลอดภัยของผู้ควบคุมข้อมูลส่วนบุคคล พ.ศ. 2565 รวมถึงหน้าที่อื่น ๆ ตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 ได้แก่

  • มาตรา 19 ขอความยินยอม (Consent) ในการเก็บรวบรวม ใช้ หรือเปิดเผยข้อมูลส่วนบุคคล เว้นแต่ PDPA หรือกฎหมายอื่นบัญญัติให้กระทำได้
  • มาตรา 23 แจ้งประกาศความเป็นส่วนตัว (Privacy Notice) (ดูตัวอย่าง)
  • มาตรา 28-29 จัดให้มีมาตรการ/นโยบาย คุ้มครองข้อมูลส่วนบุคคลที่โอนไปต่างประเทศ
  • มาตรา 30-36 จัดให้มีช่องทางการใช้สิทธิ์ของเจ้าของข้อมูลส่วนบุคคลและช่องทางการแจ้งเหตุรั่วไหลของข้อมูลส่วนบุคคล
  • มาตรา 37
    1. จัดให้มีมาตรการคุ้มครองข้อมูลส่วนบุคคล (ดูตัวอย่าง)
    2. ป้องกันไม่ให้บุคคลหรือนิติบุคคลอื่นที่ได้รับข้อมูลส่วนบุคคล ใช้หรือเปิดเผยข้อมูลส่วนบุคคลโดยปราศจากอำนาจหรือโดยมิชอบ
    3. จัดให้มีระบบตรวจสอบ เพื่อลบหรือทำลายข้อมูลส่วนบุคคลเมื่อพ้นกำหนดระยะเวลาการเก็บรักษา
    4. แจ้งเหตุละเมิดข้อมูลส่วนบุคคลตามหลักเกณฑ์และวิธีการที่คณะกรรมการคุ้มครองข้อมูลส่วนบุคคลกำหนด
  • มาตรา 39 บันทึกรายการกิจกรรมการประมวลผลข้อมูลส่วนบุคล (Record of Processing Activities)
